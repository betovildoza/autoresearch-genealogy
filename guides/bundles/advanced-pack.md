# Advanced Pack

Use this when you have a sourced tree and want to push into harder record sets.

## Who It Is For

- Your core tree has sources and remaining brick walls.
- You are comfortable reviewing AI output against original records.
- You want focused searches by archive, region, time period, or unresolved question.

## What To Prepare

- A stable `Family_Tree.md`.
- Person files or notes for the target line.
- `Open_Questions.md` with one question per research target.
- Known name variants, places, religions, migration routes, and date ranges.

## Order

1. Run [08 Open Question Resolution](../../prompts/08-open-question-resolution.md) for a single target.
2. Run [09 Local History Extraction](../../prompts/09-bygdebok-extraction.md) if you have a local history source.
3. Run [10 Colonial Records Search](../../prompts/10-colonial-records-search.md) for pre-1800 American lines.
4. Run [11 Immigration Search](../../prompts/11-immigration-search.md) for migration questions.
5. Run [02 Cross-Reference Audit](../../prompts/02-cross-reference-audit.md) after any major addition.

## Stop And Review

Stop if:

- A result depends on a same-name match without family, date, and place support.
- A local history book lacks citations for the claim you need.
- The AI skips negative results or does not explain failed searches.
- A colonial or immigration clue creates multiple possible identities.

## Human Review

Use these checks before accepting changes:

- `review-cards/08-open-question-resolution.md`
- `review-cards/09-bygdebok-extraction.md`
- `review-cards/10-colonial-records-search.md`
- `review-cards/11-immigration-search.md`
- `review-cards/02-cross-reference-audit.md`

## Expected Outputs

- Narrow research leads.
- Search logs with negative results.
- Archive-specific next steps.
- Source-backed additions, not just bigger trees.
